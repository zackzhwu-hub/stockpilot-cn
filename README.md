# A股智能分析 Agent（StockPilot-CN）项目开发文档

> **项目定位**：基于大模型多 Agent 架构的 A 股市场智能分析系统
> **文档版本**：v1.0 | 2026-04-28

---

## 一、项目概述

### 1.1 背景与痛点

A股市场拥有超过 2 亿投资者，但面临三大核心痛点：

| 痛点 | 现状 | StockPilot-CN 的解法 |
|------|------|---------------------|
| 信息过载 | 每日数千条公告、研报、新闻，散户无法有效过滤 | 多源数据聚合 + LLM 摘要提炼 |
| 分析门槛高 | 财务分析、技术面、资金面需要专业知识 | 自然语言交互，降低使用门槛 |
| 决策主观性强 | 散户缺乏系统化分析框架 | Agent 编排多维度结构化分析流程 |

### 1.2 项目目标

构建一个**多 Agent 协作**的 A 股智能分析系统，用户通过自然语言对话即可完成从个股诊断到投资决策辅助的全流程分析。

核心能力：

- **个股深度分析**：基本面 + 技术面 + 资金面 + 舆情面，四维一体
- **行业对比分析**：同行业横向对比，识别相对优势
- **研报摘要生成**：自动抓取并总结券商研报核心观点
- **风险预警**：实时监控持仓异动、公告风险、资金异常
- **投资组合诊断**：持仓健康度评估、相关性分析、仓位建议

### 1.3 目标用户

- 个人投资者（A 股散户）
- 独立分析师 / 财经自媒体
- 小型私募基金研究员

### 2.2 Agent 职责分工

| Agent | 核心职责 | 输入 | 输出 |
|-------|---------|------|------|
| **Orchestrator** | 理解用户意图、拆解子任务、调度专业 Agent、汇总分析结论 | 用户自然语言查询 | 结构化分析报告 |
| **Fundamental Agent** | 财务数据分析：营收/利润/现金流/ROE/负债率等 | 股票代码 | 财务健康度评分 + 关键指标解读 |
| **Technical Agent** | 技术面分析：K线形态、均线系统、MACD/RSI/KDJ、支撑压力位 | 股票代码 + 时间区间 | 技术面评分 + 趋势判断 |
| **Capital Agent** | 资金流向分析：主力/散户资金、北向资金、融资融券、龙虎榜 | 股票代码 | 资金面评分 + 异动提示 |
| **Sentiment Agent** | 舆情分析：新闻情感、社交媒体热度、行业政策 | 股票代码/行业 | 舆情评分 + 事件清单 |
| **Research Agent** | 研报聚合：抓取券商研报、提取核心观点、汇总目标价 | 股票代码 | 研报摘要 + 机构观点汇总 |
| **Risk Agent** | 风险评估：财务预警、退市风险、质押风险、合规风险 | 股票代码 / 持仓列表 | 风险等级 + 风险事项清单 |

### 2.3 Agent 间协作流程

以用户提问「帮我分析一下比亚迪（002594）现在能不能买」为例：

```
用户提问
  │
  ▼
Orchestrator 解析意图 → 识别为"个股综合分析"
  │
  ├─→ Fundamental Agent: 拉取比亚迪近 3 年财报，计算财务指标
  ├─→ Technical Agent: 获取近 60 日 K 线，分析技术形态
  ├─→ Capital Agent: 查询近 10 日资金流向、北向持股变动
  ├─→ Sentiment Agent: 搜索近 7 日新闻，分析舆情倾向
  ├─→ Research Agent: 汇总近 30 日研报观点和目标价
  └─→ Risk Agent: 检查质押率、商誉、减持公告等风险项
  │
  ▼
Orchestrator 汇总六维分析 → 生成结构化报告
  │
  ▼
输出：综合评分 + 各维度详细分析 + 操作建议（买入/观望/回避）
```

## 三、核心功能模块详细设计

### 3.1 基本面分析 Agent

**数据源**：

- 东方财富 / 同花顺 API：财务报表（资产负债表、利润表、现金流量表）
- 企查查 MCP（已接入）：股权结构、关联公司、经营风险
- 公司公告数据

**分析维度**：

```
盈利能力：ROE、ROA、毛利率、净利率趋势（近 3 年 + TTM）
成长能力：营收增速、利润增速、PEG
偿债能力：资产负债率、流动比率、速动比率
现金流质量：经营性现金流/净利润比率、自由现金流
估值水平：PE/PB/PS 分位数（与自身历史、同行业对比）
```

**输出格式**：

```json
{
  "stock_code": "002594",
  "stock_name": "比亚迪",
  "fundamental_score": 82,
  "metrics": {
    "roe": {"value": "18.5%", "trend": "上升", "rank_in_industry": "前 20%"},
    "pe_ttm": {"value": 25.3, "percentile_5y": "35%"}
  },
  "highlights": ["连续 4 季度营收增速超 30%", "经营性现金流大幅改善"],
  "concerns": ["应收账款周转天数同比增加 15 天"],
  "summary": "基本面整体健康，盈利能力和成长性突出..."
}
```

### 3.2 技术面分析 Agent

**数据源**：A 股实时/历史行情 API

**分析能力**：

| 分析类型 | 具体指标 |
|---------|---------|
| 趋势判断 | MA5/10/20/60/120/250 均线排列、趋势强度 |
| 动量指标 | MACD（金叉/死叉）、RSI（超买超卖）、KDJ |
| 形态识别 | 头肩顶/底、双顶/底、三角形整理、箱体突破 |
| 量价关系 | 量价配合度、放量突破/缩量回调判定 |
| 支撑压力 | 关键价位识别（整数关口、前期高低点、缺口） |

**特色功能**：自然语言解释技术指标含义，让不懂 K 线的散户也能理解。

### 3.3 资金面分析 Agent

**数据源**：

- 主力资金流向数据（东方财富）
- 北向资金数据（沪深港通）
- 融资融券数据
- 龙虎榜数据
- 大宗交易数据

**核心逻辑**：

```
IF 近 5 日主力资金持续净流入
   AND 北向资金持股比例上升
   AND 融资买入额占比 < 30%（非杠杆驱动）
THEN 资金面信号 = 偏多
```

### 3.4 舆情分析 Agent

**数据源**：

- 财经新闻（新浪财经、东方财富、财联社）
- 社交媒体（雪球、股吧）
- 政策文件（证监会公告、发改委文件）
- 行业动态

**工作流程**：

1. 关键词搜索：公司名、股票代码、高管姓名、主营产品
2. 文章抓取与清洗
3. MiMo 模型情感分析（正面/中性/负面 + 置信度）
4. 事件提取（重组、减持、业绩预告、诉讼等）
5. 影响力评估（事件对股价的潜在影响程度）

### 3.5 研报聚合 Agent

**数据源**：券商研报数据库（东方财富研报中心、慧博投研等）

**核心能力**：

- 自动抓取指定股票近 30-90 天研报
- MiMo 模型提取核心观点：买入/增持/中性/减持、目标价、核心逻辑
- 汇总机构共识与分歧
- 识别"研报密集期"（多家券商同时覆盖通常意味着拐点）

### 3.6 风险评估 Agent

**风险检查清单**：

```
□ 财务风险：连续亏损、现金流为负、商誉占净资产比 > 50%
□ 治理风险：大股东高比例质押、频繁关联交易、管理层减持
□ 合规风险：证监会处罚、交易所问询函、诉讼仲裁
□ 交易风险：ST/*ST 标识、涨跌幅异常、停牌风险
□ 退市风险：触及注册制退市指标（营收 < 1 亿、面值 < 1 元等）
□ 流动性风险：日均成交额 < 5000 万、换手率过低
```

### 3.7 Orchestrator（编排 Agent）

**核心职责**：

1. **意图识别**：解析用户自然语言，判断分析类型（个股/行业/组合/对比）
2. **任务拆解**：将复杂分析拆解为可并行的子任务
3. **Agent 调度**：按依赖关系编排 Agent 执行顺序
4. **结果汇总**：融合各 Agent 输出，消除矛盾，生成统一结论
5. **记忆管理**：维护对话上下文，支持追问和多轮交互

**调度策略**：

```
对于个股分析请求：
  Step 1 (并行): Fundamental + Technical + Capital + Risk
  Step 2 (依赖 Step 1): Sentiment（结合基本面结果筛选相关新闻）
  Step 3 (并行): Research
  Step 4 (汇总): Orchestrator 融合所有结果 → 生成报告
```

## 五、MiMo 模型使用方案

### 5.1 模型选型与 Token 消耗估算

| 使用场景 | 模型 | 日均调用量 | 单次 Token（估算） | 日均消耗 |
|---------|------|-----------|-------------------|---------|
| Orchestrator 任务理解与汇总 | GLM-5.1 | 500 次 | ~4K input + ~2K output | ~3M Tokens |
| 基本面分析解读 | GLM-5.1 | 300 次 | ~8K input + ~1.5K output | ~2.85M Tokens |
| 技术面分析解读 | Qwen 3.6 Plus | 300 次 | ~2K input + ~1K output | ~0.9M Tokens |
| 资金面分析 | Qwen 3.6 Plus | 300 次 | ~2K input + ~1K output | ~0.9M Tokens |
| 舆情情感分析 | Qwen 3.6 Plus | 1000 次 | ~1K input + ~0.3K output | ~1.3M Tokens |
| 研报摘要提取 | MiMo-V2-Pro | 200 次 | ~15K input + ~2K output | ~3.4M Tokens |
| 风险评估 | MiMo-V2-Flash | 300 次 | ~3K input + ~0.5K output | ~1.05M Tokens |
| 图表/K线形态识别 | Kimi K2.6 | 200 次 | ~2K input + ~1K output | ~0.6M Tokens |
| **合计** | | **~3100 次/日** | | **~14M Tokens/日** |

### 5.2 多模型调度策略

采用 **模型路由** 策略，根据任务特征自动选择最优模型：

| 任务特征 | 路由目标 | 原因 |
|---------|---------|------|
| 复杂推理、多步决策 | GLM-5.1 | 综合推理能力强，适合 Orchestrator 和深度分析 |
| 长文本（>10K Token） | MiMo-V2-Pro | 256K 上下文窗口，适合完整研报/年报分析 |
| 高并发、低延迟（情感/摘要） | Qwen 3.6 Plus | 响应快、成本优，适合批量处理 |
| 图像/图表理解 | Kimi K2.6 | 多模态能力，适合 K 线图、财务图表解读 |
| 轻量补充任务 | MiMo-V2-Flash | 低成本兜底 |

### 5.3 为什么选择多模型组合

1. **发挥各自优势**：不同模型在不同任务上各有特长，组合使用比单一模型效果更好
2. **成本优化**：轻量任务用小模型，复杂任务用大模型，避免"杀鸡用牛刀"
3. **冗余保障**：单一模型 API 故障时，可自动切换到备选模型
4. **中文场景全覆盖**：GLM、MiMo、Qwen、Kimi 均对中文深度优化，与 A 股分析场景天然契合

## 九、项目创新点

1. **多 Agent 证券分析协作**：业内首个基于多 Agent 架构的 A 股开源分析系统，每个 Agent 专注一个分析维度
2. **自然语言全流程交互**：从数据查询到分析报告，全程自然语言，零学习成本
3. **多模型智能路由**：根据任务特征自动选择最优模型（GLM 推理、MiMo 长文本、Qwen 高并发、Kimi 多模态），发挥各自优势
4. **MCP 工具链复用**：企查查、MinerU 等已部署 MCP 直接嵌入 Agent 工具链，减少重复开发
5. **开源 + 可自部署**：MIT 协议开源，用户可自行部署，数据不上传第三方
